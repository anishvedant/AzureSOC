# T1003.001 — LSASS Memory Credential Dumping

## MITRE ATT&CK
- **Tactic:** Credential Access
- **Technique:** T1003.001 — OS Credential Dumping: LSASS Memory
- **Severity:** HIGH

## What Is This Attack?
LSASS (Local Security Authority Subsystem Service) is the Windows process that handles authentication. When you log into a Windows machine, your password hash (and sometimes plaintext password) is stored in LSASS memory. Attackers use tools like Mimikatz to read this memory and extract credentials, which they then use to move laterally through the network.

## Red Team — How To Execute
```powershell
# Option 1: Atomic Red Team
Invoke-AtomicTest T1003.001 -TestNumbers 1

# Option 2: Manual (ProcDump — signed Microsoft tool, harder to detect)

# Option 3: Task Manager
# Right-click lsass.exe > Create Dump File
```

## Blue Team — What To Look For

### Sysmon Event ID 10 (Process Access)
The key indicator is any process accessing lsass.exe with suspicious GrantedAccess values:
- `0x1010` — PROCESS_QUERY_LIMITED_INFORMATION | PROCESS_VM_READ
- `0x1038` — common Mimikatz pattern
- `0x1F0FFF` — PROCESS_ALL_ACCESS (most obvious)

### Sentinel KQL Detection
```kql
Event
| where Source == "Microsoft-Windows-Sysmon"
| where EventID == 10
| parse EventData with * '<Data Name="TargetImage">' TargetImage '</Data>' *
| parse EventData with * '<Data Name="SourceImage">' SourceImage '</Data>' *
| parse EventData with * '<Data Name="GrantedAccess">' GrantedAccess '</Data>' *
| where TargetImage endswith "lsass.exe"
| where GrantedAccess in ("0x1010", "0x1038", "0x1F0FFF")
```

### Splunk SPL Detection
```spl
index=idx_sysmon EventCode=10 TargetImage="*lsass.exe"
| search GrantedAccess IN ("0x1010", "0x1038", "0x1F0FFF")
| table _time host SourceImage TargetImage GrantedAccess
```

## SOAR — Automated Response
1. **Isolate endpoint** via Defender for Endpoint API
2. **Disable compromised user** accounts in Entra ID
3. **Collect investigation package** from endpoint
4. **Create incident ticket** in Jira/ServiceNow
5. **Notify SOC channel** via Teams/Slack
6. **Block source IP** if attack came from network

## Evidence to Collect
- Sysmon Event ID 10 logs showing the accessing process
- Memory dump of the suspected tool process
- Network connections made by the attacking process (Sysmon Event ID 3)
- Parent process chain (what launched the tool?)
